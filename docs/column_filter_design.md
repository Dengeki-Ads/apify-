# GAS 不要列自動削除スクリプト 設計書（改訂版）

## 1. 目的

Apifyで取得したデータを書き込んだGoogleスプレッドシートに対して、
必要な列のみを残し不要な列を自動削除する。
Webhook受信によるデータ書き込み完了後に自動実行される。

---

## 2. 前提条件

| 項目 | 内容 |
|------|------|
| 実行環境 | Google Apps Script（GAS） |
| トリガー | 既存の `doPost()` 内、`appendDataRows()` の直後に呼び出し |
| 対象シート | `data` シート（固定） |
| ヘッダー行 | 1行目（固定） |
| 列指定方法 | ホワイトリスト方式（残す列のヘッダー名を指定し、それ以外を削除） |
| 確認ダイアログ | なし（完全自動実行） |

---

## 3. 設定管理

既存の設計に合わせ、**スクリプトプロパティ**で管理する。

| プロパティキー | 型 | 説明 | 例 |
|---|---|---|---|
| `COLUMNS_TO_KEEP` | String（カンマ区切り） | 残す列のヘッダー名リスト。先頭の `fetched_at`, `run_id` は常に残るため指定不要 | `name,views,likes,shares` |

`config.gs` に以下のヘルパーを追加する:

- `getColumnsToKeep()` — スクリプトプロパティ `COLUMNS_TO_KEEP` をパースして配列として返す。常に保持する列（`fetched_at`, `run_id`）を自動付与する。

---

## 4. 処理フロー

```
doPost()（既存・webhook.gs）
  └─ appendDataRows() でデータ書き込み完了
       └─ filterColumns() を呼び出し
            ├─ data シートを取得
            ├─ ヘッダー行を読み取り
            ├─ COLUMNS_TO_KEEP に含まれない列番号を特定
            ├─ 列番号を降順ソート
            ├─ 後ろの列から順に削除
            └─ ログ出力
```

---

## 5. 関数一覧

### 5-1. getColumnsToKeep()（config.gs に追加）

| 項目 | 内容 |
|------|------|
| 役割 | スクリプトプロパティ `COLUMNS_TO_KEEP` をパースし、常に保持する列を付与して返す |
| 引数 | なし |
| 戻り値 | `string[]` — 残す列名の配列 |

### 5-2. filterColumns()（新規ファイル columns.gs）

| 項目 | 内容 |
|------|------|
| 役割 | data シートからホワイトリスト外の列を削除する |
| 引数 | なし |
| 戻り値 | なし |
| 呼び出し元 | `doPost()` 内、`appendDataRows()` の直後 |

**処理ステップ**

1. `getSheet('data')` で対象シートを取得する
2. シートにデータがない場合（`lastRow === 0`）はスキップログを出力して終了
3. ヘッダー行（1行目）の全セルを取得する
4. `getColumnsToKeep()` で保持する列名リストを取得する
5. ヘッダーを走査し、保持リストに含まれない列の列番号を収集する
6. 保持リストに含まれるがシート上に存在しない列名を警告ログに出力する
7. 削除対象が0件の場合はスキップログを出力して終了
8. 削除対象の列番号を降順にソートする
9. 後ろの列から順に `sheet.deleteColumn()` で削除する
10. 削除完了した列名と件数を成功ログに出力する

---

## 6. エラーハンドリング仕様

既存のエラー処理パターン（Gmail通知 + logシート記録）に合わせる。

| ケース | 対応 |
|--------|------|
| `COLUMNS_TO_KEEP` が未設定 | 列削除をスキップ（WARNログ出力）。データ書き込み自体は正常完了とする |
| data シートにデータがない | SKIPログを出力して処理を終了 |
| 保持リストの列名がシートに存在しない | WARNログに列名を出力し、他の列の削除は続行 |
| 削除対象が0件 | SKIPログを出力して処理を終了 |
| 削除処理中にエラー発生 | `doPost()` の既存 try-catch で捕捉され、Gmail通知 + logシート更新が行われる |

**注意:** `COLUMNS_TO_KEEP` が未設定の場合はエラーにせずスキップする。
これにより、この機能を使わない運用でも既存動作に影響しない。

---

## 7. 削除順序に関する仕様

列を前から順に削除すると、削除のたびに後続列の番号がずれてバグになる。
これを防ぐため、**削除対象の列番号を降順にソートし、後ろの列から順に削除する**。

```
例：B列(2)・D列(4)・F列(6)を削除する場合
  誤った順序：2 → 4 → 6（2を消すとDが3列目にずれる）
  正しい順序：6 → 4 → 2（後ろから消すため前の列番号に影響しない）
```

---

## 8. 組み込み箇所

既存の `webhook.gs` の `doPost()` 内、`appendDataRows(headers, rows)` の直後に1行追加するのみ:

```javascript
appendDataRows(headers, rows);
filterColumns();          // ← この1行を追加
```

---

## 9. ログ出力仕様

GASの `Logger.log()` に出力する。

| レベル | プレフィックス | 出力タイミング |
|--------|---------------|---------------|
| 正常完了 | `[COLUMNS OK]` | 削除完了時（削除列名・件数を含む） |
| スキップ | `[COLUMNS SKIP]` | データなし・削除対象0件・COLUMNS_TO_KEEP未設定 |
| 警告 | `[COLUMNS WARN]` | 保持リストの列名がシートに存在しない |
